Post 07

Study, Document, Implement, Evaluate

Quatro fases para fazer o agente desacelerar o suficiente para entender o que está fazendo.

O fluxo parece simples: estudar, documentar, implementar, avaliar. Simples o bastante para caber numa linha. Difícil o bastante para salvar uma tarde.

A tentação é sempre pular para o código. O agente quer ser útil. O humano quer ver progresso. A issue parece clara. A vida é curta. Vamo.

Aí vem o YODA e fala: calma. Primeiro Study. Depois Document. Só então Implement. E no final Evaluate, porque entregar sem conferir é basicamente jogar a mochila por cima do muro e chamar de logística.

O Flow é a placa de “devagar” na descida. Ninguém gosta dela até perceber o tamanho da curva.

Esse post é sobre a disciplina de avançar uma fase por vez e deixar rastros suficientes para saber o que aconteceu, quando aconteceu e por quê.

O que este post vai cobrir

  • O papel de cada fase do YODA Flow.
  • Por que o avanço é explícito e humano-guiado.
  • Como o Evaluate fecha a história com resultado auditável.
