Post 09

Empacotar uma teimosia

Uma coisa é funcionar no meu repositório. Outra é sair de casa sem quebrar a casa dos outros.

Quando o YODA começou a virar pacote, o problema deixou de ser só processo. Agora tinha distribuição, instalação, atualização, checksum, rollback, metadata, versão e aquele frio na barriga de publicar algo que alguém pode executar.

Foi aí que o projeto precisou ficar menos artesanal. O pacote tinha que levar o framework, mas preservar o projeto host. Tinha que atualizar, mas permitir voltar. Tinha que ser fácil, mas não mágico demais.

Instalador de uma linha é bonito. Versão pinada é mais responsável. Os dois precisavam conviver.

Empacotar o YODA foi transformar uma mania local em algo que sobrevive fora do meu colo.

Esse post conta a parte menos glamourosa e mais necessária: fazer a odisséia virar artefato distribuível.

O que este post vai cobrir

  • package.py, manifest e changelog.
  • latest.json, checksum e instalador.
  • Upgrade, rollback e limites do pacote dentro de outro projeto.
