O que morreu no caminho

Nem toda idéia ruim nasce feia. Algumas nascem elegantes, ganham nome e depois atrapalham com muita convicção.

O YODA teve conceitos que pareciam úteis. lightweight. Campo agent. Entry points no YAML. Defaults demais em template. ID no front matter. Init mexendo em arquivo host como se estivesse em casa.

Alguns morreram rápido. Outros precisaram de issue, migração, release e um pequeno velório técnico.

Remover conceito é mais difícil que criar. Criar dá sensação de avanço. Remover parece admitir derrota. Mas muitas vezes é o contrário: é o projeto ficando mais honesto.

A maturidade do YODA não veio só do que entrou. Veio muito do que saiu.

Esse post é sobre desapegar. Sobre perceber que uma regra pode ser tecnicamente possível e ainda assim ruim para o fluxo.

O que este post vai cobrir

  • Conceitos removidos entre 0.1.x e 0.4.0.
  • Por que metadata demais vira ruído.
  • Como remoções melhoraram a clareza do contrato.
