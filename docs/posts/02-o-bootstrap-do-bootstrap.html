O bootstrap do bootstrap

A fase mais honesta do projeto: o framework ainda não existia, mas já precisava ajudar a criar o próprio framework.

Tem uma ironia bonita em usar um processo para construir o processo. Bonita depois. Na hora, parecia mais uma daquelas idéias que começam com “vai ser rápido”.

No começo, não havia script suficiente, não havia pacote, não havia fluxo redondo. Havia specs, TODOs, arquivos temporários, decisões sendo registradas no calor da corrida e uma fé moderada de que aquilo tudo ia se organizar.

Foi o período do bootstrap-legacy/. A gaveta histórica. O lugar onde ficaram os artefatos que não são mais o caminho oficial, mas contam como a caminhada realmente aconteceu.

O bootstrap foi o YODA de chinelo: ainda sem uniforme, mas já tentando ensinar o agente a não atravessar a rua sem olhar.

Esse post conta essa fase meio improvisada, meio necessária. Porque antes de ter script determinístico, alguém precisa descobrir quais regras merecem virar script.

O que este post vai cobrir

  • Como o projeto começou como meta-implementação.
  • Por que o legado foi preservado em vez de apagado.
  • O valor de registrar decisões mesmo quando o processo ainda está torto.
