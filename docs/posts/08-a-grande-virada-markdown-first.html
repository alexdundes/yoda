Post 08

A grande virada Markdown-first

O nome dizia YAML. A realidade, teimosa como sempre, apontou para o Markdown.

YODA significa YAML-Oriented Documentation & Agents. Bonito. Organizado. Com cara de arquitetura. Só que, conforme o projeto amadureceu, ficou claro que YAML demais estava tentando contar história.

E YAML é ótimo para metadado. Mas péssimo para contexto vivo. Ele não gosta de respirar. Não gosta de nuance. Não gosta de frase atravessada que explica por que uma decisão foi tomada.

A issue Markdown começou como camada narrativa e terminou como fonte operacional de verdade. O front matter ficou com o mínimo necessário. O resto foi para onde deveria estar: texto legível, versionado, auditável.

A virada Markdown-first foi o YODA admitindo: “meu nome é YAML, mas meu coração é Markdown”.

Esse post conta a transição da 0.3.0, quando o projeto parou de duplicar informação e começou a confiar mais no arquivo que o humano realmente lê.

O que este post vai cobrir

  • ID derivado do filename.
  • Dependências em front matter, sem duplicação no corpo.
  • Flow log embutido na issue Markdown.
