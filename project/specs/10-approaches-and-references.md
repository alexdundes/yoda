# Approaches and references

## Documentation and design

- Documentation-Driven Development (DocDD)
- README-Driven Development (RDD)
- Docs-as-Code
- Design-First / Contract-First for APIs
- ADR (Architecture Decision Records)
- Literate Programming / Notebooks (exploration)

## Relation to YODA

- DocDD and Docs-as-Code support documentation as source of truth.
- RDD reinforces the README as the entry point.
- Design-First helps with contracts and integrations.
- ADRs record technical decisions.
- Notebooks can be used for exploration and prototypes.
