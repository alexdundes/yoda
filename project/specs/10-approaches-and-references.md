# Approaches and references

## Documentation and design

- `DocDD (Documentation-Driven Development)`
- `RDD (README-Driven Development)`
- `Docs-as-Code`
- `Design-First / Contract-First`
- `ADRs (Architecture Decision Records)`
- `Literate Programming / Notebooks`
- `Agentic Design Patterns`

## Relation to YODA

- `DocDD` and `Docs-as-Code` support documentation as the source of truth.
- `RDD` reinforces the README as the entry point.
- `Design-First / Contract-First` helps with contracts and integrations.
- `ADRs` record technical decisions.
- `Notebooks` can be used for exploration and prototypes.
- `Agentic Design Patterns` provide a vocabulary for planning, routing, reflection, and human-in-the-loop patterns.

## Details per influence

The details of each influence are in `project/specs/influences/` (one file per topic).
